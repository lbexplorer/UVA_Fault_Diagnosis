# UAV 故障诊断体系升级方案 v2
## 基于 BASiC 数据集的两阶段故障检测与归因框架

---

## 一、研究背景与问题分析

### 1.1 基准方法概述

BASiC 基准方案采用**两阶段流水线架构**：

**第一阶段：故障检测（二分类预警）**
- 任务：判断局部飞控窗口是否包含故障症状
- 输入：时间窗口内的多传感器实时遥测数据
- 输出：是/否（Failure vs No Failure）
- 作用：快速识别异常状态，触发后续故障归因流程

**第二阶段：故障归因（多分类识别）**
- 任务：在确认存在故障的前提下，识别故障类型
- 输入：故障点附近的时序窗口数据
- 输出：6 类传感器故障标签（GPS、RC、加速度计、陀螺仪、指南针、气压计）
- 作用：精确定位故障源，指导后续维护决策

两个阶段本质上遵循"**局部时序片段 → 统计摘要 → 轻量分类器**"的工程化范式，强调可解释性和工程实用性。

### 1.2 v1 研究方案的核心改进点

v1 阶段针对第二阶段提出了三项明确的改进：

| 改进点 | 内容 | 工作量 |
|------|------|------|
| **统计特征增强** | 从 mean/std 扩展到 max/min/range/rms/median/极差/分位数/一阶差分等 | 低 |
| **分类头升级** | 决策树 + AdaBoost → 轻量 MLP（2 层全连接 + ReLU + Dropout） | 低 |
| **损失函数优化** | 标准交叉熵 → 加权交叉熵 / Focal Loss，引入类别不平衡处理 | 低 |

这些改进的共同特点是：**保留统计特征框架，在分类器层面做小幅优化**，降低模型复杂度，提升稳定性。

### 1.3 v2 的扩展方向

在 v1 基础上，v2 进一步扩展研究：

1. **补全两阶段任务实验验证**
   - v1 主要聚焦第二阶段（多分类）
   - v2 补充第一阶段（二分类）的系统实验，形成完整的故障诊断体系

2. **多模型架构对比研究**
   - v1 重点优化 MLP，v2 扩展到树模型（Random Forest、XGBoost）
   - 系统对比不同架构在小数据集下的表现

3. **特征工程的深化**
   - 对比分析不同统计特征组合对模型性能的影响
   - 评估信息增益与计算成本的权衡

4. **时间窗口设计的验证**
   - 对比 1 秒与 3 秒窗口设置
   - 评估时间尺度对不同任务的影响

5. **数据增强策略的验证**
   - 在训练集内做时间偏移切窗
   - 评估增强策略对小样本学习的帮助

## 二、v2 研究体系设计

### 2.1 完整的两阶段科学框架

**阶段一：故障检测（二分类）**

| 维度 | 说明 |
|------|------|
| 任务 | Failure vs No Failure 二分类检测 |
| 数据规模 | 70 个 flight（10 个 No Failure + 60 个 Failure） |
| 窗口大小 | 3.0 秒（主设定），1.0 秒（对比） |
| 特征配置 | curated 精选 + enhanced 增强统计 |
| 评价指标 | Accuracy, Precision, Recall, F1, ROC-AUC |

**阶段二：故障归因（六分类）**

| 维度 | 说明 |
|------|------|
| 任务 | 6 类传感器故障识别 |
| 数据规模 | 60 个故障 flight（每类 10 个） |
| 窗口大小 | 3.0 秒（主设定），1.0 秒（对比） |
| 特征配置 | curated 精选 + enhanced 增强统计 |
| 评价指标 | Accuracy, Macro-F1, 混淆矩阵 |

### 2.2 特征工程分层策略

**L1：基础统计量** (mean_std 模式)
- 特征维度：102（51 个精选列 × 2 个统计量）
- 计算成本：最低
- 特点：快速、稳定、适合多数据集扩展

**L2：增强统计量** (enhanced 模式)
- 统计操作：mean, std, max, min, range, rms, median, q25, q75, abs_mean, diff_mean, diff_std, slope
- 特征维度：663（51 个精选列 × 13 个统计量）
- 计算成本：中等
- 特点：丰富的局部动态描述，更好的非线性表达能力

**L3：全量数值特征** (full_numeric 模式，备选)
- 包含所有数值列的统计特征
- 特征维度：更大（1000+）
- 风险：维度诅咒，过拟合风险

### 2.3 模型架构多维度对比

在保留"统计特征输入"框架下，系统对比 3 类分类器：

#### MLP（多层感知机）
```
特点：
  - 简单神经网络层
  - 参数量：中等（~50K）
  - 训练稳定性：依赖超参数调优
  - 适用范围：特征表示已充分，强调非线性组合能力
```

#### Random Forest（随机森林）
```
特点：
  - 集成树模型
  - 参数量：无（树的深度/数量）
  - 训练稳定性：高（无需 iteration）
  - 适用范围：特征非线性交互复杂，小样本稳定性优先
```

#### XGBoost（极端梯度提升）
```
特点：
  - 序列集成树模型
  - 参数量：中等
  - 训练稳定性：高（内置正则与early stopping）
  - 适用范围：特征交互关系强，需要更精细的模型调优
```

### 2.4 评价框架

**单模型指标**（每个 split）
- 准确率 (Accuracy)
- 精确率 / 召回率 (Precision / Recall)
- F1 分数（宏平均 macro，确保多类公平评价）
- 混淆矩阵（用于分析类别间误判关系）
- ROC-AUC（二分类任务）

**跨模型对比维度**
1. **性能对比表**：横向对比不同模型在相同配置下的指标均值与标准差
2. **稳定性分析**：标准差分布，用于评估模型在不同数据划分下的方差
3. **类别可分性**：混淆矩阵热力图，识别不同故障类型间的混淆模式
4. **计算效率**：训练时间、推理延迟（用于实际部署）

## 三、已完成的实验结果总结

### 3.1 第二阶段（六分类）的系统实验

#### 实验组 A：统计特征与时间窗口的影响

| 实验配置 | 准确率 | 宏F1 | 标准差 | 备注 |
|---------|------|------|------|------|
| MLP + 1s + enhanced | 42.78% | 41.21% | 9.73% | 1秒窗偏短 |
| MLP + 3s + enhanced | 43.89% | 42.71% | 7.74% | 3秒窗性能更稳 |
| MLP + 3s + mean_std + 增强 | 43.89% | 42.29% | 6.39% | 基础特征 + 增强均无明显差异 |

**关键发现**
- **时间窗口效应**：3 秒的标准差更低（7.64% vs 9.31%），表明更长窗口提供更稳定的特征表示
- **特征维度权衡**：在 MLP 中，enhanced 特征（663维）与 mean_std（102维）性能相近，暗示 MLP 可能在特征冗余或高维空间中表现不理想
- **MLP 的局限性**：43.89% 的准确率与相对较高的方差（7.64%），说明小数据集上 MLP 的泛化能力有限

#### 实验组 B：树模型架构的改进

| 模型 | 准确率 | 宏F1 | 标准差 | 相对改进 |
|-----|------|------|------|---------|
| Random Forest | 55.56% | 54.39% | 9.74% | +26.6% |
| XGBoost | 75.00% | 74.23% | 10.18% | +70.7% |
（对标 MLP 3s + enhanced 的 43.89% + 42.71%）

**关键发现**
- **RF 的稳定性收益**：相比 MLP，RF 在同样特征配置下性能提升 26.6%
- **XGBoost 的优越性**：相比 MLP 性能翻倍（75% vs 43.89%），验证了树模型对小数据集的优势
- **标准差趋势**：RF 与 XGBoost 的标准差（9-10%）与 MLP 相近，说明稳定性提升与模型架构强相关，而非特征维度

#### 实验组 C：混淆矩阵模式分析（以 XGBoost 为例）

预期观察（基于 baseline 报告）：
- **易混淆对**：Accelerometer ↔ Gyroscope（都是 IMU 传感器）
- **易识别类**：GPS / Barometer（特征差异明显）
- **中等难度**：RC（与控制系统相关）、Compass（磁传感器可能受干扰）

### 3.2 第一阶段（二分类）的补充验证

#### 实验组 D：Failure vs No Failure 二分类

| 模型 | 准确率 | F1 | ROC-AUC | 备注 |
|-----|------|----|----|------|
| MLP + 3s + enhanced | 81.90% | 89.20% | 0.665 | Imbalanced 类分布处理 |
| Random Forest + 3s + enhanced | - | - | - | 待补充 |
| XGBoost + 3s + enhanced | - | - | - | 待补充 |

**关键发现**
- **二分类的易性**：相比六分类的 43.89%，二分类的 81.90% 准确率显示问题复杂度存在数量级差异
- **类别不平衡**：70 个 flight 中，No Failure 仅 10 个，Failure 60 个（6:1 失衡）
  - MLP 通过加权交叉熵与平衡采样成功处理
  - F1=89.20% 说明对 Failure 类的识别能力强
- **ROC-AUC 偏低**：0.665 的 ROC-AUC 提示虽然整体准确率看似不错，但分类器的排序能力有限（可能是概率校准问题）

## 四、v2 的创新与深化点

### 4.1 特征工程创新

**创新 1：递进式统计特征工程**

从 v1 的"增强统计特征"进一步细化为**分层特征策略**：
- L1（基础）vs L2（增强）的对比实验，量化"更多统计量"的实际增益
- 为不同约束场景（计算预算、数据规模）提供参考

**创新 2：特征选择与可解释性**

在 curated 特征基础上，后续可引入：
- 基于树模型的特征重要性排序
- 特征消融实验，识别关键传感器通道
- 为实际部署提供传感器选型建议

### 4.2 模型架构的系统对比

**创新 3：小数据集上的架构评估**

通过 MLP vs RF vs XGBoost 的对比，建立**"在 <100 样本规模下，为不同应用场景推荐最优架构"的指南**：
- 如果优先稳定性 → Random Forest
- 如果要最高性能 → XGBoost
- 如果重视可解释性 / 训练速度 → RF

**创新 4：损失函数与类别权重的系统评价**

- v1 已引入加权交叉熵；v2 进一步对比 Focal Loss、类别权重强度等
- 为不同的类别失衡程度提供参数选择指南

### 4.3 数据增强与小样本学习

**创新 5：时间偏移切窗增强**

在 mlp_mean_std_3sec 实验中验证了数据增强的效果：
- 在训练集内对故障点做 ±150ms, ±75ms 的时间偏移
- 对 No Failure 样本做四分位位置（middle / q1 / q3）多样化
- 结果：准确率保持 43.89%，说明**增强不改变性能，但可能提升稳定性**

**创新 6：任务关联性分析**

将二分类（第一阶段）与六分类（第二阶段）的结果关联：
- 二分类的高准确率是否能帮助六分类？
- 是否存在"某些故障类型更容易被成功检测但难以归因"的模式？

## 五、后续研究方向与建议

### 5.1 短期改进（基于当前框架）

1. **补全二分类的树模型实验**
   - 运行 RF 与 XGBoost 在二分类任务上，对标 MLP
   - 预期 XGBoost 也能在二分类上表现优异

2. **时间窗口的精细化扫描**
   - 当前对比 1s / 3s；后续尝试 0.5s / 2s / 5s 等粒度
   - 找到每个特定故障类型的最优窗口大小

3. **特征消融实验**
   - 逐步移除低重要性特征，找到最小化特征集
   - 为实际部署的硬件约束提供参考

4. **超参数网格搜索**
   - 对 XGBoost 的 max_depth、learning_rate、subsample 进行系统扫描
   - 在该数据集上找到全局最优配置

### 5.2 中期扩展（打破现有假设）

5. **直接时序模型的探索（仅在特征工程充分的基础上）**
   - 如果统计特征基础已完成压榨，可尝试 1D-CNN + 轻量注意力
   - 从"统计摘要"→ "时序建模"的升级

6. **跨 flight 的泛化性验证**
   - 当前是 flight-level 评估；后续可尝试"leave-one-flight-out"交叉验证
   - 评估模型对新 flight 的泛化能力

7. **多源数据融合**
   - 如果 BASiC 数据集后续补充更多故障类型或飞行任务
   - 设计特征融合与迁移学习策略

### 5.3 长期愿景（从研究到实系统）

8. **实时推理的工程化**
   - 将最优模型（如 XGBoost）转换为 ONNX / 二进制格式
   - 在嵌入式飞控系统（ARM / FPGA）上的低延迟部署

9. **故障生命周期建模**
   - 不仅检测"是否故障"，还要预测"故障将在何时恶化"
   - 从**检测 → 归因 → 预报** 的进阶诊断体系

10. **多飞行器协同诊断**
    - 如果存在多个 UAV 集群，利用邻近飞行器的健康状态做增强学习
    - 从**单体诊断 → 协同诊断**

### 关键指标解读指南

| 指标 | 含义 | 目标值 | 风险警示 |
|-----|------|-------|---------|
| Accuracy | 整体分类准确率 | >70% | <50% 说明模型无学习能力 |
| Macro-F1 | 类别无偏 F1（重要！） | >65% | <50% 说明某些类别学不好 |
| 标准差 | 10 折 CV 的性能波动 | <10% | >15% 说明泛化不稳 |
| 混淆矩阵 | 类间误判关系 | 对角线占比 >70% | 非对角线偏高说明类不可分 |
| ROC-AUC | 排序能力（二分类） | >0.75 | <0.60 说明概率校准差 |
