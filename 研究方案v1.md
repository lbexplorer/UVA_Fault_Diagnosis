一、问题分析
baseline 整体是两阶段流水线：
第一阶段：判断“是否将发生故障”，属于二分类预警。
第二阶段：在已经确定“发生了故障”的前提下，进一步判断到底是哪一种传感器故障，属于六分类识别。

第二阶段的本质不是“发现故障”，而是“故障归因”。它回答的问题是：
已知当前这段飞行数据附近存在故障迹象，那么故障来源是
GPS、RC、Accelerometer、Gyroscope、Compass、Barometer 中的哪一类？

baseline 在这一阶段没有再用 Transformer，而是用了一个很保守、很工程化的方案：
1.以故障点附近的一小段时间窗为对象；
2.对每个特征做均值 mean + 标准差 std统计聚合；
3.然后把这个统计特征向量输入 AdaBoost + Decision Tree 做六分类。

所以它本质上是：
“局部时序片段 → 统计摘要 → 轻量分类器”











二、研究任务应该如何定义
任务定义
输入：
来自 BASiC Processed Data 的时序片段，这些片段围绕已知故障发生时刻构造，且标签属于 6 类传感器故障之一。

输出：
六分类标签：
1)GPS failure √
2)Remote control failure √
3)Accelerometer failure √
4)Gyroscope failure√
5)Compass failure 
6)Barometer failure

主方案：
“统计特征增强 + 类别不平衡处理 + 轻量 MLP 分类头”

改进点 1：统计特征增强
1. Baseline 原有做法：
Baseline 在故障识别阶段，对故障点附近的时序窗口按特征维度计算 均值（mean） 和 标准差（std），再将这些统计量拼接成特征向量输入分类器。也就是说，baseline 使用的是较基础的二阶统计摘要来表示局部故障片段。
2. 改进方案：
在 baseline 的 mean/std 基础上，增加少量容易实现的统计特征，如 最大值、最小值、极差、均方根（RMS）、一阶差分均值/标准差 等，用更丰富的局部统计信息描述不同传感器故障的动态差异。该改进不改变整体流程，只是对输入分类器的特征表达做小幅增强。
3. 实现难度与工作量评价：
该改进实现难度低，主要工作是补充特征提取代码，基本不涉及复杂模型设计。可复现性高，也便于后续通过消融实验验证各类统计特征的贡献。

改进点 2：分类头轻量替换

1. Baseline 原有做法：
Baseline 在得到统计特征后，采用 Decision Tree + AdaBoost 进行六分类识别。这种方法实现简单、可解释性较强，但对更复杂的类别边界和多特征组合关系建模能力有限。

2. 改进方案：
将 baseline 的树模型分类器替换为一个 轻量 MLP 分类头，如两层全连接网络加 ReLU 与 Dropout。这样既保留“统计特征输入—轻量分类输出”的整体框架，又能提升模型对非线性特征组合的表达能力，属于小幅结构优化。

3. 实现难度与工作量评价：
该改进实现难度较低，工作量主要集中在模型训练与参数调节，整体远低于重新设计复杂时序网络。可复现性较好，适合作为 baseline 分类器的直接替换方案。

改进点 3：损失函数与类别学习策略优化

1. Baseline 原有做法：
Baseline 论文重点描述了第二阶段的统计聚合与 AdaBoost 决策树分类流程，但没有突出针对类别学习难度差异进行专门优化，整体上更偏向标准分类流程。

2. 改进方案：
在训练阶段引入 加权交叉熵 或 Focal Loss，使模型在训练过程中对较难区分或较易混淆的类别给予更多关注；必要时可结合简单的类别权重设置，提高模型对少数困难类别的识别稳定性。该改进不改变数据处理和模型主体，只优化训练目标。

3. 实现难度与工作量评价：
该改进实现难度很低，通常只需替换损失函数或增加类别权重配置即可完成。工作量小、复现方便，是一种风险很低但较容易带来稳定收益的优化方式。


预期结果：
在不依赖额外数据、不引入大规模预训练的前提下，实现稳定的 6 类传感器故障识别，并在 accuracy、macro-F1 或分类稳定性上优于或不劣于 baseline 第二阶段。

为保证任务定义清晰，本研究采用 flight-level classification 作为主要评价口径，即：
每个故障 flight 对应一个分类样本； 
模型输出一个六分类结果； 
以该结果与该 flight 的真实故障类型进行比较。

本研究主要采用以下指标评价分类性能：
Accuracy
Macro Precision
Macro Recall
Macro F1
Confusion Matrix

其中，Macro-F1 用于更公平地衡量 6 类故障上的整体识别能力；混淆矩阵用于分析不同故障类别之间的误判关系。baseline 论文中也给出了第二阶段的混淆矩阵结果，并指出 Accelerometer 与 Gyroscope 存在一定混淆，因此本研究也将重点关注这些类别对之间的可分性。

三、样本构造规则
1 数据来源
本研究直接使用 BASiC 数据集中的 Processed Data。该部分数据已经完成时间序列化、插值和平滑处理，并在整合后的单表 CSV 中包含统一的 Status 标签，适合直接用于时序建模与故障分类研究。

2 样本定义
本研究聚焦第二阶段“故障识别与分类”任务，因此仅使用 6 类故障飞行样本，不纳入 no-failure 类别。每个故障 flight 依据真实故障时刻截取局部时间窗，并将该时间窗视为一个分类样本，对应一个故障类别标签。baseline 第二阶段本质上也是围绕故障点附近数据做统计聚合后进行六分类。

3 时间窗构造方式

以真实故障时刻为中心，截取固定长度的局部时序片段用于分类。具体设置如下：

主设定：1 秒时间窗
对比设定：3 秒时间窗

其中，1 秒窗对应 baseline 第二阶段测试时“故障点前后各 500ms”的局部数据聚合思路；3 秒窗用于考察更长局部动态信息是否有助于分类性能提升。

4 样本数量组织原则
为简化问题并避免样本定义过于复杂，本研究采用：
每个故障 flight 提取 1 个主样本 作为基础实验设定；
如后续需要扩充训练样本，可仅在训练集内对故障点附近做轻微时间偏移切窗，形成辅助训练样本；
测试集始终保持“每个 flight 仅 1 个样本”，以保证评价口径统一。

5 数据划分原则
训练集与测试集严格按照 flight 层面划分，而不是按窗口随机划分。这样可避免同一 flight 的不同时间窗同时出现在训练集和测试集，从而降低数据泄漏风险。考虑到 baseline 在 BASiC 上采用的是分层 70:30 划分（每类 10 个 flight 中 7 个训练、3 个测试），本研究可沿用这一比例，并在此基础上增加交叉验证作为补充。
三、备选方案
备选方案一：只改损失函数与采样策略
可以做一个极稳的备选：
保留 baseline 的统计特征思路，只把分类器稍作替换或不替换：
mean + std 不变 
决策树/AdaBoost 不变，或换成 Random Forest / XGBoost 
引入： 
class weight 
focal loss（如果换成神经网络） 
balanced sampler 

特点：这条线最容易做，适合当保底实验。
即使主方案效果一般，也可以写出：
在 baseline 第二阶段框架下，通过类别代价敏感学习提升了故障分类稳定性。
风险
创新较小，但胜在稳。
备选方案二：轻量注意力增强的 1D-CNN 分类模型
这是比 MLP 稍微“深度学习一点”的版本，但仍然可控。
做法
直接对故障点附近的原始时序窗做分类：
输入：T × d 
网络： 
o1D Conv 
o1D Conv 
oSE 注意力 或 简单通道注意力 
oGlobal Average Pooling 
oFC 输出 6 类 
为什么可行
要求优先考虑“轻量注意力增强、分类头改进、时序窗口设计”，这个方案正好符合。而且 SE 模块很轻，不会像 Transformer 那么重。
风险：
比主方案稍高，因为：
需要直接训练时序模型 
小数据集上可能不如统计特征稳 
所以作为第二备选，不作为主线。
